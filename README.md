## Hi there 👋
# I'm Utkarsh Jha

Backend & full-stack engineer, CS 2025. Based in India.

I work mostly on the server side — APIs, data pipelines, and the parts that decide whether something holds up once real people use it. I'm comfortable in C++, live on Linux, and care about how a system actually runs, not just whether it ships.

### Live projects

- **[Knowledge OS](https://evolve-snowy.vercel.app/)** ([code](https://github.com/utkarshjha1407/mluemind)) — research fields mapped as problems instead of lists of papers: how a field evolved, what won, what's still open. 14,400 papers and ~379k citations from OpenAlex, processed by a Python engine and served as a static React app with an in-browser retrieval agent. No backend or API keys at runtime.
- **[InterviewAI](https://reinterviewai.vercel.app/)** ([code](https://github.com/utkarshjha1407/v1)) — mock interviews with questions pulled from your real GitHub repos, or a topic you pick. FastAPI and Groq backend, 0–10 scoring with written feedback, rate-limited and JWT-protected.
- **[GaadiGuru](https://gaadiguru.vercel.app)** ([code](https://github.com/utkarshjha1407/mrwhite)) — a car-services app, built solo from a Figma prototype to a deployed Next.js 14 + Supabase product, with role-based access enforced through row-level security.

### How I work

- I default to free-tier, self-hostable infrastructure. All three projects run at roughly $0/month, with no paid services in the hot path.
- I'd rather own and understand the whole stack than wire managed services together and hope.
- I'd rather a system flag what it's unsure about than return a confident wrong answer.
