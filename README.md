Engineering philosophy

I like systems that

- fail loudly instead of silently
- cost less than coffee to run
- don't require fifteen managed services to return JSON
- explain why they failed
- stay maintainable after the excitement wears off

Simple beats clever.

Predictable beats magical.

Shipping beats endlessly redesigning.
